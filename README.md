                ## Resumo do Projeto

Este projeto consiste em uma aplicação React Native para gerenciamento de lembretes de medicamentos, construída com Expo.  A aplicação permite aos usuários agendar lembretes, visualizar lembretes existentes e gerenciar as notificações.  A persistência de dados é feita via AsyncStorage.  O projeto utiliza diversas bibliotecas para UI, navegação, gerenciamento de data/hora e notificações.  Testes são feitos usando Jest e Babel.


## Propósito e Descrição do Projeto

Criar um aplicativo móvel que auxilie os usuários a gerenciar seus horários de medicação através de lembretes e notificações.  O aplicativo visa melhorar a adesão ao tratamento medicamentoso, fornecendo uma interface simples e intuitiva para agendamento e gerenciamento de lembretes.


## Regras de Negócio

* Os lembretes de medicamentos devem conter informações como nome do medicamento, horário e frequência de administração.
* O aplicativo deve solicitar permissões de notificação ao usuário.
* As notificações devem ser enviadas em horários específicos, levando em conta a frequência e o horário de início definidos pelo usuário.
* O aplicativo deve persistir os dados dos lembretes mesmo após o fechamento e reabertura do aplicativo.
* O aplicativo deve exibir uma lista dos lembretes agendados.
* O aplicativo deve permitir que o usuário exclua lembretes.


## Arquitetura

A aplicação utiliza uma arquitetura baseada em componentes React Native. Os principais componentes incluem:

* **App.js:** Componente principal que gerencia a navegação entre as telas.
* **Home.js:** Tela principal que exibe a lista de lembretes.
* **Add.js:** Tela para adicionar novos lembretes.
* **Alarm.js:** Componente para renderizar informações de um lembrete individual.
* **showAlert.js:** Função para exibir mensagens de alerta ao usuário.


## Pipeline

* **Desenvolvimento:**  O código é escrito e testado localmente usando o Expo Go e/ou emuladores.
* **Build:**  O aplicativo é construído para as plataformas Android e iOS usando o Expo CLI.
* **Testes:**  Testes unitários são executados utilizando o Jest.
* **Deploy:** O aplicativo é publicado nas lojas de aplicativos (Google Play Store e Apple App Store).  Potencialmente usa serviços como Expo EAS para automação de builds e deploy.
